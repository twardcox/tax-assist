## SOW+ Review Checklist

When presenting the SOW+ for approval, include this checklist. The PM/PO must confirm each before responding `Output Approved.`:

- [ ] Objectives are clear and measurable
- [ ] Scope is bounded (in-scope and out-of-scope defined)
- [ ] Risks are assessed and mitigation strategies identified
- [ ] Stakeholders and success criteria are documented
- [ ] Architecture/tech stack inventory is complete (if applicable)
- [ ] No critical gaps or conflicting requirements

---

## 4. Upfront Clarifying Questions

**Before generating artifacts, confirm:**

| Question                            | Why It Matters                |
| ----------------------------------- | ----------------------------- |
| "Spreadsheet, markdown, or both?"   | Prevents rework               |
| "How many tasks need full specs?"   | Avoids over-engineering       |
| "Runnable code or just structure?"  | Clarifies scaffolding depth   |
| "Which framework doc is canonical?" | Resolves conflicting guidance |
| "Which artifacts matter most?"      | Enables prioritization        |
