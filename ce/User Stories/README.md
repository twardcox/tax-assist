# User Stories

User story creation phase: **one Jira story per user story, with all of its acceptance criteria included** — never split ACs across tickets, and don't create "component tasks" that should really be separate user stories.

## Contents

- [task-template.md](task-template.md) — Task tickets for **change orders or targeted work** when a full user story isn't appropriate.

## Where story structure is defined

- Story format and Given/When/Then ACs: see [Epics/epic-template.md](../Epics/epic-template.md) (stories are specified inside epics) and [PRD/milestone-prd-template.md](../PRD/milestone-prd-template.md) §3.
- Creating tickets from specs: `/create-prd-tickets`; ad-hoc tickets: `/new-ticket` (task/bug quick path, or user story via the change workflow — see [Change Workflow/](../Change%20Workflow/README.md)).

Shared phase rules (checkpoint protocol, `Output Approved.` gate): [SHARED-GUIDELINES.md](../SHARED-GUIDELINES.md).
